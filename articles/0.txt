Windows 如何配置环境变量？
为什么需要配置环境变量，这篇文章简单告诉你。
环境变量,cmd,windows
2020-02-11
打开CMD随便输入一个随机字符串，按下回车会产生下面的提示
<pre><code>
'abcpagkpakh' 不是内部或外部命令，也不是可运行的程序
或批处理文件。
</code></pre>
如果我们尝试输入"ping"、"mspaint"等字符串再按回车，发现前者会在窗口上输出如下内容
<pre>
C:\Users\i>ping

用法: ping [-t] [-a] [-n count] [-l size] [-f] [-i TTL] [-v TOS]
            [-r count] [-s count] [[-j host-list] | [-k host-list]]
            [-w timeout] [-R] [-S srcaddr] [-c compartment] [-p]
            [-4] [-6] target_name

选项:
    -t             Ping 指定的主机，直到停止。
                   若要查看统计信息并继续操作，请键入 Ctrl+Break；
                   若要停止，请键入 Ctrl+C。
                   ...
</pre>
而后者会打开Windows默认的画图程序。

CMD能正确识别到这两个命令，就是因为它们包含在系统的环境变量"PATH"里面，这个PATH变量的本质就是路径
CMD扫描路径发现路径下存在一个可执行文件"ping.exe"，就不会报错，Windows上对于可执行文件可以省略后缀名
比如"ping"实际上对应的是"ping.exe"文件，如果不是可执行文件则必须输入文件全名，比如"services.msc"。

如何将路径添加到PATH变量呢？在CMD里面输入下面的命令，回车可进入高级系统设置，在底部有"环境变量"按钮
<pre>
<code>SystemPropertiesAdvanced</code>
</pre>在"系统变量"中的变量一栏可以找到里面名为"PATH"的变量，直接双击它即可编辑，在右上角有新建按钮
点击它，再把你需要添加的值输入或粘贴进去就行了。

新建环境变量需要粘贴什么值进去呢？就是前面提到的路径，把有可执行文件的路径粘贴进去就行，以 Java 为例

为什么输入 java 提示不是内部或外部命令，也不是可运行的程序？

这是因为没有告诉CMD你的 Java 可执行文件在什么地方，你现在只需要找到 java.exe 的路径加入到环境变量"PATH"就不会出错了
例如我的Java路径
<code>C:\Program Files\Java\jdk-24\bin\</code>
加入环境变量之后，必须关闭原来的CMD，开启一个新的CMD窗口，新窗口会加载你修改的变量，从而找到对应的可执行文件

如果仍然找不到则意味着路径写错了，必须是一个存在的路径，不能是一个文件，如果是压缩包则必须解压。

注意事项: 必须将路径加入到PATH变量，CMD才能正常识别，PATH变量的本质就是一堆路径拼接在一起，用";"符号隔开不同的路径，
且符号必须是英文状态下的，千万不要输入成中文的"；"符号
这是原始的PATH变量示例:
<code>
C:\Windows;C:\Windows\System32;
</code>
在Windows10及以上系统可以忽略这些，直接按上面的一个一个新建变量即可，无需手动拼接。
